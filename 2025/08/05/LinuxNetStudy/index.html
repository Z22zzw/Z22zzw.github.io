分层标准不同

有分七层 应用层、表示层、会话层、传输层、网络层、数据链路层、物理层

有分四层 应用层、传输层、网络层、链路层

TCP/IP协议 是最基本的协议 protocol

将电脑连接

数据链路层：解读01的组合

使用Ethernet以太网协议

帧Frame数据包分为 Head 和 Data

标头head固定18字节bit

MAC地址：网卡地址，想当于确认发送位置和接受位置的地址

  • 所以网卡地址是独一无二的，48位二进制 由12个16进制表示

既然有了MAC地址，那么该如何确认双方的MAC地址呢？

  • 这里就需要用到广播了，也就是广发送所有用户，然后通过比对标头，由计算机自身判断是否接受。

1号计算机向 2 号计算机发送一个数据包，同一个子网络的 3 号、4号、5号计算机都会收到这个包。它们读取这个包的“标头”，找到接收方的 MAC 地址，然后与自身的 MAC 地址相比较，如果两者相同，就接受这个包，做进一步处理，否则就丢弃这个包。这种发送方式就叫做**“广播”**（broadcasting）。

网络层

因为广播太原始了，如果隔着太远，对于全球互联网想要发送数据包给子网的所有用户太不现实。所以需要用到网络层。

规定网络地址的协议，叫做 IP 协议。它所定义的地址，就被称为 IP 地址。

目前，广泛采用的是 IP 协议第四版，简称 IPv4。这个版本规定，网络地址由 32 个二进制位组成。

  • 如果能连接IPv4公网那么就能当服务器了

IP地址可以从0.0.0.0到255.255.255.255

IP地址总可以分为两个部分，前一部分代表网络号，后一部分代表主机号

那么前一部分后一部分到底是以什么为界限来判断前后呢？

  • 这里就需要用子网掩码

所谓“子网掩码”，就是表示子网络特征的一个参数。它在形式上等同于 IP 地址，也是一个 32 位二进制数字，它的网络部分全部为1，主机部分全部为0，并且1和0分别连续。

比如，IP 地址 172.16.254.1，如果已知网络部分是前 24 位，主机部分是后 8 位，那么子网络掩码就是 11111111.11111111.11111111.00000000，写成十进制就是 255.255.255.0。
